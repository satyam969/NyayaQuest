# NyayaQuest 🏛️

**NyayaQuest** is a highly specialized, fast, and open-source AI legal assistant powered by localized Semantic Retrieval-Augmented Generation (RAG).

Currently indexed exclusively with the **Bharatiya Nyaya Sanhita (BNS) 2023**, NyayaQuest leverages state-of-the-art embedding models to understand complex legal queries and returns highly accurate statutes along with reasoning generated by LLaMA-3.

## 🚀 Features
- **Structural Chunking**: Parses legal PDFs fundamentally by Section (e.g., "Section 103.") to preserve strict legal boundaries.
- **Semantic Search**: Uses `BAAI/bge-small-en-v1.5` to calculate semantic similarity, matching natural language scenarios to dense legal texts.
- **High-Performance LLM**: Powered by Groq's API running `llama-3.3-70b-versatile` for lightning-fast legal reasoning.
- **Multi-Query Formulation**: Automatically re-writes user queries into formal *legalese* before searching the database to maximize hit accuracy.
- **Memory**: Features a localized Redis connection for persistent conversational history, equipped with an automatic fallback to local RAM if a Redis server isn't found on the host machine.

## 🛠️ Tech Stack
- **Python Framework**: LangChain
- **Frontend**: Streamlit
- **Vector Database**: ChromaDB (Local)
- **Embeddings**: SentenceTransformers (`BAAI/bge-small-en-v1.5`)
- **LLM**: Groq (LLaMA-3)
- **Caching**: Redis

## ⚙️ Setup & Installation

**1. Install Dependencies**
This project requires `uv` to manage the environment.
```bash
uv sync
```

**2. API Keys**
Create a `.env` file in the root directory and add your Groq key:
```ini
GROQ_API_KEY=your_groq_api_key_here
```

**3. Ingest Data (Optional)**
The Vector database is pre-built, but to reset or add more laws:
1. Place target PDFs in `data/legal_pdfs/`
2. Run the ingestion script:
```bash
uv run python src/ingest_structural.py
```

**4. Run the Application**
```bash
uv run streamlit run app.py
```
